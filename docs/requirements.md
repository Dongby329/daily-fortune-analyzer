## 🧭 用户旅程（Happy Path）

```
用户打开页面
    ↓
输入出生年月日（必填）+ 时间（选填）
    ↓
点击「查看今日运势」
    ↓
    ├─ 第一屏：综合评分（大数字）+ 一句话总结
    ├─ 第二屏：分类运势（事业/感情/财运/健康）
    ├─ 第三屏：性格解析（基于星座+八字）
    ├─ 第四屏：人际契合（今日宜近/宜远人群特质）
    └─ 第五屏：今日幸运物（颜色/数字/方位）
    ↓
用户点击「分享」→ 生成运势卡片 → 分享到社交平台
```

## ⚠️ 关键假设与风险

| # | 假设/风险 | 影响 | 应对策略 |
|---|----------|------|---------|
| R1 | **算命算法的准确性与逻辑性** | ⭐⭐⭐ 核心风险。用户说"不够准"就不会再来 | 星座部分用权威数据源；八字部分用传统排盘算法，每项结论附简短推演逻辑 |
| R2 | **用户留存** | ⭐⭐⭐ 单次使用后无回头 | 每日运势需要"每天不一样"，不能用户第二天打开看到差不多的内容 |
| R3 | **命理数据源的可靠性** | ⭐⭐ 传统命理缺少统一标准 | 采用公开经典算法（如《渊海子平》排盘规则），注明算法依据 |
| R4 | **内容合规性** | ⭐⭐ 算命类内容可能涉及迷信争议 | 定位为"娱乐参考"+"传统文化科普"，页面底部加免责声明 |
| R5 | **星座 × 传统命理融合难度** | ⭐⭐ 两套体系逻辑不同 | 分别计算后通过加权融合公式输出综合评分，不强行统一逻辑 |
