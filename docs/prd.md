## 1. 产品概述

### 一句话

**输入出生信息，获得星座 × 传统命理融合的每日运势分析报告。**

### 要解决的问题

现有星座 App 太泛（只看太阳星座），传统算命 App 太土（黄历风格、功能堆砌）。用户想要一个**精准、有逻辑、好看的**每日运势工具——既能看星座，又能结合八字，还能分享。

### 目标用户

| 用户 | 场景 | 频次 |
|------|------|------|
| 18-35 岁年轻人 | 早上起来看一眼今日运势，决定穿搭/心态 | 每天 |
| 星座爱好者 | 想要比普通星座 App 更个性化的运势 | 每天 |
| 传统文化好奇者 | 对八字生肖有兴趣但觉得传统 App 太复杂 | 偶尔 |

### 产品目标

| 目标 | 衡量 |
|------|------|
| 用户 1 分钟内完成"输入 → 看到综合评分" | 加载时间 < 2s |
| 次日留存 ≥ 40% | 内容每天有变化 |
| 分享率 ≥ 15% | 分享卡片好看 |

### 2.2 P0 功能详细规格

#### F-01 出生信息采集

**用户故事**：作为用户，我要输入自己的出生日期和时间，以便获得个性化的今日运势。

**前置条件**：无

**主流程**：
1. 用户打开页面 → 看到表单
2. 选择出生年月日（date picker，必填）
3. 选择出生时间（time picker，选填）
4. 点击"查看今日运势" → 进入加载 → 跳转结果页

**异常处理**：

| 异常 | 处理 |
|------|------|
| 未填日期直接提交 | 浏览器原生 required 拦截 |
| 日期为未来日期 | 提交时校验，提示"请输入正确的出生日期" |
| 年份早于 1900 | 提示"暂不支持 1900 年以前的日期" |

**交互说明**：
- 日期和时间用浏览器原生 input[type=date/time]，移动端自动唤起原生选择器
- 时间输入框下方显示提示文案："填写后可解锁八字精算"
- "随机体验"按钮一键填充随机合法生日

**验收标准**：
- [x] 日期必填，未填时无法提交
- [x] 时间选填，不填也能提交
- [x] 未来日期被拦截
- [x] 随机按钮一键填充合法值
- [x] 提交后显示加载动画，1.5s 后跳转结果页

---

#### F-02 星座运势计算

**用户故事**：作为用户，我要看到基于我星座的今日运势，以便了解今天在事业/感情/财运/健康方面的走势。

**前置条件**：用户已提交出生月日

**主流程**：
1. 从出生月日查找星座（如 7/23 → 狮子座）
2. 获取当前日期 → 计算今日星象（简化模型：基于日期的数学推算）
3. 根据星座属性（火/土/风/水）与今日星象相位，计算星座运势分 (0-100)
4. 生成四个维度的运势文案（事业/感情/财运/健康）

**星座日期对照表**：
```
白羊 3/21-4/19  金牛 4/20-5/20  双子 5/21-6/21
巨蟹 6/22-7/22  狮子 7/23-8/22  处女 8/23-9/22
天秤 9/23-10/23 天蝎 10/24-11/22 射手 11/23-12/21
摩羯 12/22-1/19 水瓶 1/20-2/18  双鱼 2/19-3/20
```

**异常处理**：无，星座判定是确定性的。

**验收标准**：
- [x] 任意月日输入正确判定星座
- [x] 边界日期（如 7/22 vs 7/23）准确分属不同星座
- [x] 不同日期产生不同的运势分数
- [x] 分数在 0-100 范围内

---

#### F-03 生肖运势计算

**用户故事**：作为用户，我要看到我生肖的今日运程。

**前置条件**：用户已提交出生年份

**主流程**：
1. 出生年份 mod 12 → 生肖（如 1995 → 猪）
2. 获取当前日期的地支 → 与生肖地支关系分析
3. 计算生肖运势分 (0-100)

**生肖对照**：
```
鼠 牛 虎 兔 龙 蛇 马 羊 猴 鸡 狗 猪
子 丑 寅 卯 辰 巳 午 未 申 酉 戌 亥
```

**异常处理**：年份不在生肖表中的极端值（<1900）已在 F-01 拦截。

**验收标准**：
- [x] 任意合法年份正确判定生肖
- [x] 不同日期产生不同的生肖运势

---

#### F-04 八字排盘

**用户故事**：作为填写了出生时间的用户，我要看到自己的八字排盘结果，以获得更精准的运势分析。

**前置条件**：用户已提交出生年月日时（四者齐全）

**主流程**：
1. 年柱：出生年份 → 天干地支（如 1995 → 乙亥）
2. 月柱：出生年月 → 天干地支（用节气判定，非公历月份）
3. 日柱：出生年月日 → 天干地支（万年历推算）
4. 时柱：出生日 + 时 → 天干地支
5. 日主 = 日柱天干
6. 十神分析：日主与其他三柱天干的关系

**降级处理**：
- 无出生时间 → 只算年柱+月柱+日柱（三柱），不排时柱
- 无出生时间 → 八字运势权重降低，星座和生肖权重提高

**验收标准**：
- [x] 有时柱 → 完整四柱八字 + 十神
- [x] 无时柱 → 三柱 + 提示"填写时间可解锁完整八字"
- [x] 八字运势分在 0-100 范围内

---

#### F-05 综合评分

**用户故事**：作为用户，我第一眼就要看到一个直观的综合评分，快速了解今日整体运势。

**前置条件**：F-02/F-03/F-04 计算完成

**主流程**：
1. 权重配置：
   - 有出生时间：星座 40% + 八字 40% + 生肖 20%
   - 无出生时间：星座 60% + 生肖 40%
2. 加权求和 → 综合评分 (0-100)
3. 分数映射等级：

| 分数 | 等级 |
|------|------|
| 90-100 | 极佳 |
| 75-89 | 良好 |
| 60-74 | 平稳 |
| 40-59 | 稍低 |
| 0-39 | 低迷 |

4. 选取最高分和最低分维度 → 组合生成一句话总结

**交互说明**：
- 评分以圆环 + 大数字展示，结果页第一屏
- 一句话总结在评分下方，用引号包裹

**验收标准**：
- [x] 评分在 0-100 范围
- [x] 等级映射正确
- [x] 有/无出生时间使用不同权重
- [x] 一句话总结与各维度分数逻辑一致

---

#### F-06 分类运势

**用户故事**：作为用户，我要看到事业/感情/财运/健康四个维度的详细运势。

**显示内容**：
- 每个维度的分数 + 等级标记
- 进度条可视化
- 1-2 句针对性的建议文案

**文案要求**：
- 不重复模板化——每个维度文案由算法动态组合（维度 + 分数段 + 星座/八字特征）
- 举例："今日适合主动汇报工作，领导认可度高"（事业 · 高分 · 狮子座特征）
- 低分维度给建设性建议，不恐吓用户

**验收标准**：
- [x] 四个维度全部展示
- [x] 进度条动画流畅
- [x] 文案与分数逻辑一致（高分 = 积极，低分 = 提醒）

---

#### F-07 性格解析

**用户故事**：作为用户，我要看到基于星座和八字的性格分析，了解自己的内在特质。

**计算逻辑**：
1. 星座性格模板（12 种基础性格）
2. 八字日主性格（10 种日主性格：甲/乙/丙/丁/戊/己/庚/辛/壬/癸）
3. 交叉对比：
   - 星座 + 日主一致倾向 → 增强描述（如 狮子+丙火 = 双重火象）
   - 星座 + 日主冲突 → 互补描述（如 巨蟹+庚金 = 外柔内刚）

**显示内容**：
- 星座卡片 + 日主卡片（并排）
- 2-3 段性格描述文案
- 3-5 个性格标签（chip 样式）

**验收标准**：
- [x] 星座和八字日主都正确显示
- [x] 性格描述与两者的交叉分析一致
- [x] 无出生时间时，仅显示星座性格 + 提示"填写时间可解锁八字性格"

---

#### F-08 人际契合

**用户故事**：作为用户，我要知道今天与哪些特质的人相处更合拍、与哪些人需要保持距离。

**计算逻辑**：
1. 今日星象 → 影响人际氛围
2. 今日日柱地支 → 与不同五行人格的契合度
3. 输出：宜近人群特质（3-4 个标签）+ 宜远人群特质（2-3 个标签）

**显示内容**：
- 宜近区：绿色标记，含简短理由
- 宜远区：琥珀色标记，含简短理由

**验收标准**：
- [x] 每日动态变化（不同日期不同推荐）
- [x] 宜近/宜远都有具体特质标签，不笼统

---

#### F-09 幸运物

**用户故事**：作为用户，我要知道今天的幸运颜色、数字和方位。

**计算逻辑**：
1. 幸运色：基于星座 + 日柱五行，选择今日有利的颜色
2. 幸运数字：基于日柱干支的数理推算
3. 幸运方位：基于今日地支的方位

**显示内容**：三列网格，颜色用色块展示，数字和方位用大字展示。

**验收标准**：
- [x] 三要素全部展示
- [x] 每日有变化（不同日期不同推荐）

## 3. 非功能需求

### 性能
- 页面加载 < 1s（纯静态 HTML，CDN 引入 Tailwind）
- 运势计算 < 200ms（纯客户端 JS 运算）
- 分享卡片生成 < 500ms

### 兼容性
- Chrome / Safari / Edge 最近 2 个大版本
- iOS Safari 14+ / Android Chrome 90+
- 移动端（<768px）和桌面端（≥768px）响应式

### 可用性
- 从打开到看到结果 ≤ 3 步
- 必填项只有 1 个（出生日期）
- 所有计算在浏览器本地完成，无需网络
- 不收集/不上传任何用户数据

### 设计原则
- 黑色主题（#000），白色文字
- 极简风格：细边框 + 留白 + 单色系统
- 字体层级：大数字（评分）→ 标题 → 正文 → 辅助文字
- 星空表情背景缓慢滚动，不干扰阅读

## 4. 数据需求

### 静态数据（内置于代码中）

| 数据 | 内容 | 来源 |
|------|------|------|
| 星座日期表 | 12 星座 × 起止日期 | 公开天文数据 |
| 生肖对照表 | 年份 mod 12 → 生肖 | 公开 |
| 干支表 | 天干 10 × 地支 12 + 60 甲子 | 公开万年历 |
| 节气表 | 24 节气近似日期 | 公开 |
| 文案模板库 | 运势描述片段（按维度×分数段） | 自编 |

### 动态数据（通过算法推算）

| 数据 | 方式 |
|------|------|
| 今日星象 | 基于当前日期的数学简化模型 |
| 今日日柱 | 万年历算法推算 |
| 综合评分 | 加权融合算法 |

### 不存储的数据
- 用户出生日期 / 时间
- 任何用户身份信息
- 任何计算结果

## 7. 隐私与安全

| 数据 | 存储 | 上传？ |
|------|------|--------|
| 出生日期/时间/密码 | localStorage | ❌ 永不 |
| 用户名/ID/头像 | localStorage | ❌ 永不 |
| API Key | localStorage（用户profile） | ❌ 永不 |
| 星座/元素/性别 | — | ⚠️ 仅发送给LLM API（不可追溯） |
| 聊天消息 | — | ⚠️ 仅发送给LLM API |
| 穿搭搜索关键词 | — | 仅用于生成平台搜索链接 |

- 所有用户数据仅存浏览器本地
- 无服务器/无数据库/无第三方追踪
- LLM API 调用仅发送匿名化的运势特征
